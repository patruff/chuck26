Create `training/merge_and_push.py` -- a standalone script that merges the trained LoRA adapter into FP16 base model weights and pushes the result to HuggingFace Hub.

The script MUST follow the exact patterns from 05-RESEARCH.md Pattern 4. Specifically:

**Configuration section (top of file):**
- `BASE_MODEL = "unsloth/Qwen3-32B-unsloth-bnb-4bit"`
- `ADAPTER_PATH = "/workspace/dpo-output/final-adapter"` (output of train_dpo.py)
- `MERGED_OUTPUT_DIR = "/workspace/merged-model"`
- `HUB_MERGED_REPO = "patruff/chuckles-qwen3-32b-dpo"`
- `MAX_SEQ_LENGTH = 2048`

**Workflow:**
1. Check that `ADAPTER_PATH` exists on disk (exit with clear error if not -- means train_dpo.py hasn't been run)
2. Check that `HF_TOKEN` env var is set (exit with error if not)
3. Check available disk space on /workspace/ and warn if less than 150GB (merge needs ~130GB)
4. Load base model with `FastModel.from_pretrained()` (same params as training script)
5. Load the trained adapter: `model.load_adapter(ADAPTER_PATH)`
6. Save merged model locally FIRST: `model.save_pretrained_merged(MERGED_OUTPUT_DIR, tokenizer, save_method="merged_16bit")` -- local save as safety net before Hub push
7. Verify merged model files exist on disk (check for safetensors files in MERGED_OUTPUT_DIR)
8. Push merged model to Hub: `model.push_to_hub_merged(HUB_MERGED_REPO, tokenizer, save_method="merged_16bit", token=os.environ["HF_TOKEN"])`
9. If push_to_hub_merged fails (known bug #3146 -- sometimes only pushes README), fall back to manual upload using `huggingface_hub.HfApi().upload_folder()`
10. Print success message with Hub URL

**Critical details:**
- NEVER use `save_method="merged_4bit"` -- 4-bit merge degrades quality (decision locked in STATE.md)
- The merge step will download FP16 base model (~65GB) behind the scenes -- this is expected behavior (Unsloth bug #3633)
- Include a comment about the FP16 re-download behavior so the user is not surprised
- All paths must be under /workspace/

**Module docstring** explaining:
- Prerequisites: train_dpo.py must have completed successfully
- Disk space requirements: ~130GB free on /workspace/ (65GB FP16 download + 65GB merged output)
- Expected runtime: 30-60 minutes (mostly FP16 download + merge computation)

Do NOT:
- Import from chuckles_prime package
- Use `FastLanguageModel`
- Use `merged_4bit` save method
- Hardcode HF tokens

Create `training/validate_merge.py` -- a standalone script that validates the merged model produces comparable outputs to the adapter-loaded model.

The script MUST follow 05-RESEARCH.md Pattern 5 and satisfy EXP-04. Specifically:

**Configuration section:**
- `BASE_MODEL = "unsloth/Qwen3-32B-unsloth-bnb-4bit"`
- `ADAPTER_PATH = "/workspace/dpo-output/final-adapter"`
- `MERGED_MODEL_PATH = "/workspace/merged-model"` (or load from Hub: `"patruff/chuckles-qwen3-32b-dpo"`)
- `MAX_SEQ_LENGTH = 2048`
- A flag `USE_HUB_MERGED = False` (set True to load merged model from Hub instead of local)

**Test prompts (at least 5):**
```python
TEST_PROMPTS = [
    "Create a phonetically-sound parody of: 'The Shawshank Redemption'",
    "Create a phonetically-sound parody of: 'Pulp Fiction'",
    "Create a phonetically-sound parody of: 'The Godfather'",
    "Create a phonetically-sound parody of: 'Jurassic Park'",
    "Create a phonetically-sound parody of: 'Fight Club'",
]
```

**System prompt** -- use the same system prompt from the project (a brief version):
```python
SYSTEM_PROMPT = "You are a comedy writer who creates funny parody titles. Given a movie, book, or show title, create a phonetically similar parody that sounds like the original when spoken aloud."
```

**Workflow:**
1. Load base model + adapter (same as training, with `load_adapter`)
2. For each test prompt:
   a. Format using `tokenizer.apply_chat_template()` with `enable_thinking=False`
   b. Generate with `model.generate(max_new_tokens=256, temperature=0.7)`
   c. Decode and store result
3. Print all adapter model outputs
4. Unload adapter and load merged model (either local or from Hub based on flag):
   - For local: use `AutoModelForCausalLM.from_pretrained(MERGED_MODEL_PATH)` with `torch_dtype=torch.float16`
   - For Hub: same but with Hub repo name
5. For each test prompt, generate with merged model using same parameters
6. Print all merged model outputs
7. Print side-by-side comparison table
8. Check that outputs are non-empty and not identical to a base-model default response
9. Print a PASS/FAIL summary: PASS if all merged outputs are non-empty and contain actual parody content (not just the prompt echoed back)

**Important details:**
- Use `enable_thinking=False` consistently for all template applications
- Use the same generation parameters (temperature, max_new_tokens) for both models
- The outputs will NOT be identical (quantization differences) -- the goal is "semantically comparable" not "exact match"
- Include a comment explaining this: "Outputs may differ slightly due to quantization. Check for semantic quality, not exact match."

**Module docstring** explaining:
- Prerequisites: Both train_dpo.py and merge_and_push.py must have completed
- What it validates: That merged model quality is comparable to adapter-loaded model
- Expected output: Side-by-side comparison + PASS/FAIL

Do NOT:
- Import from chuckles_prime package
- Use `FastLanguageModel`
- Expect exact output match between adapter and merged models
- Hardcode HF tokens
