<task type="auto">
  <name>Task 2: Create standalone DPO training script</name>
  <files>training/train_dpo.py</files>
  <action>
Create `training/train_dpo.py` -- the standalone DPO training script for RunPod. This is the most critical file in Phase 5.

The script MUST follow the exact patterns from 05-RESEARCH.md. Specifically:

**Configuration section (top of file):**
- `MODEL_NAME = "unsloth/Qwen3-32B-unsloth-bnb-4bit"` (pre-quantized 4-bit)
- `DATASET_NAME = "patruff/chuckles-dpo"` (existing Hub dataset)
- `OUTPUT_DIR = "/workspace/dpo-output"` (network volume!)
- `HUB_ADAPTER_REPO = "patruff/chuckles-qwen3-32b-dpo-adapter"` (Hub destination)
- `MAX_SEQ_LENGTH = 2048`
- LoRA config: `r=32, lora_alpha=32, lora_dropout=0`

**Model loading:**
- Use `FastModel` (NOT `FastLanguageModel` -- FastModel is the current API)
- `FastModel.from_pretrained()` with `load_in_4bit=True, full_finetuning=False`
- `FastModel.get_peft_model()` with target_modules: q_proj, k_proj, v_proj, o_proj, gate_proj, up_proj, down_proj
- `use_gradient_checkpointing="unsloth"` (Unsloth's optimized variant)

**Dataset loading and verification:**
- `load_dataset(DATASET_NAME, split="train")`
- Print dataset info: num_rows, column_names, sample row[0]
- Add a verification step that checks the first tokenized sample does NOT contain `<think>` blocks (Pitfall 2 from research). Print the formatted template of the first sample using `tokenizer.apply_chat_template(dataset[0]["prompt"], tokenize=False, add_generation_prompt=True, enable_thinking=False)` and assert no `<think>` in the output.

**Training configuration (DPOConfig):**
- `num_train_epochs=3`
- `per_device_train_batch_size=2`
- `gradient_accumulation_steps=4` (effective batch size = 8)
- `learning_rate=5e-6`
- `beta=0.1` (DPO temperature)
- `bf16=True` (required for Unsloth)
- `logging_steps=10`
- `save_steps=100` (checkpoint every 100 steps)
- `save_total_limit=3` (keep last 3 checkpoints)
- `max_length=2048`
- `max_prompt_length=512`
- `warmup_ratio=0.1`
- `optim="paged_adamw_8bit"` (prevent optimizer OOM spikes)
- `report_to="none"` with a comment to set to "wandb" if configured
- `loss_type="sigmoid"` (standard DPO loss)

**DPOTrainer setup:**
- `ref_model=None` (PEFT-aware: disables adapter for reference logits, zero extra VRAM)
- `processing_class=tokenizer` (current TRL API, NOT the older `tokenizer=` kwarg)

**Post-training save:**
- Save adapter locally: `model.save_pretrained_merged("/workspace/dpo-output/final-adapter", tokenizer, save_method="lora")`
- Push adapter to Hub: `model.push_to_hub_merged(HUB_ADAPTER_REPO, tokenizer, save_method="lora", token=os.environ["HF_TOKEN"])`
- Print confirmation messages

**Script structure:**
- Module docstring explaining usage, requirements, and RunPod assumptions
- `if __name__ == "__main__":` guard
- Inline comments for every non-obvious decision (WHY not just WHAT)
- Include hyperparameter tuning comments: "If rewards/accuracies quickly reaches 1.0, reduce epochs. If stuck at 0.5, increase learning rate."

Do NOT:
- Import anything from the chuckles_prime package (standalone script)
- Use `FastLanguageModel` (legacy API)
- Use `tokenizer=` parameter (old TRL API, use `processing_class=` instead)
- Hardcode HF tokens (use `os.environ["HF_TOKEN"]`)
- Save to any path outside /workspace/
