**Class: `OpenAICompatibleModel(smolagents.Model)`**

```python
import os
from openai import OpenAI
from smolagents import Model
from smolagents.models import ChatMessage, MessageRole
```

Constructor `__init__(self, model_name, api_base_url, api_key_env_var, **kwargs)`:
1. Call `super().__init__(model_id=model_name, **kwargs)` -- this sets `self.model_id`
2. Read API key from `os.environ.get(api_key_env_var)`. If not set, raise `ValueError(f"Environment variable {api_key_env_var} is not set. Export it before using the model.")`
3. Create `self.client = OpenAI(api_key=api_key, base_url=api_base_url)`
4. Store `self.api_key_env_var = api_key_env_var` for error messages

Method `generate(self, messages, stop_sequences=None, response_format=None, tools_to_call_from=None, **kwargs) -> ChatMessage`:
1. Convert incoming messages to OpenAI format. Messages can be:
   - `dict` with "role" and "content" keys -> pass through
   - Objects with `.role` and `.content` attributes (ChatMessage) -> convert role enum to string: `str(msg.role.value)` for the role, `msg.content or ""` for content
   - Anything else -> wrap as `{"role": "user", "content": str(msg)}`
2. Handle role mapping for smolagents-specific roles:
   - `MessageRole.TOOL_CALL` -> map to `"assistant"` role
   - `MessageRole.TOOL_RESPONSE` -> map to `"user"` role
   - All other roles -> use `str(msg.role.value)` directly (covers "system", "user", "assistant")
3. Build `completion_kwargs`:
   - `model`: `self.model_id`
   - `messages`: converted messages list
   - Merge `self.kwargs` (from Model base class, contains defaults like max_tokens, temperature)
   - Merge `**kwargs` (call-specific overrides)
   - If `stop_sequences` provided, add as `stop` key
4. Call `self.client.chat.completions.create(**completion_kwargs)`
5. Extract `content = response.choices[0].message.content`
6. Return `ChatMessage(role=MessageRole.ASSISTANT, content=content, tool_calls=None, raw=response)`

Important notes:
- Do NOT handle `tools_to_call_from` -- CodeAgent generates tool calls as Python code in the text response, not via the OpenAI tools API. Accept the parameter in the signature but ignore it.
- Do NOT handle `response_format` for now -- accept parameter, ignore it. Can add later if needed.
- Do NOT catch API errors. Let `openai.APIError` and subclasses propagate up. The caller (smolagents) handles retries.

**Factory function: `create_model(config) -> OpenAICompatibleModel`**

```python
def create_model(config) -> OpenAICompatibleModel:
    """Create a model from an AppConfig instance.

    Args:
        config: AppConfig with model_name, api_base_url, api_key_env_var fields

    Returns:
        Configured OpenAICompatibleModel ready for use with smolagents agents
    """
    return OpenAICompatibleModel(
        model_name=config.model_name,
        api_base_url=config.api_base_url,
        api_key_env_var=config.api_key_env_var,
        max_tokens=4096,
        temperature=0.7,
    )
```

Also add a convenience function for quick testing:

```python
def test_model_connectivity(model: OpenAICompatibleModel) -> str:
    """Send a simple prompt to verify the model connection works.

    Returns the model's response text, or raises on failure.
    """
    response = model([{"role": "user", "content": "Say 'hello' and nothing else."}])
    return response.content
```
