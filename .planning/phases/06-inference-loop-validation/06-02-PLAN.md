  <files>training/validate_inference.py</files>
  <action>
Create `training/validate_inference.py` -- a standalone Python script that compares parody generation quality between the fine-tuned model and base Qwen3-32B. Follow the pattern of `training/validate_merge.py` (configuration constants at top, main() function, clear structured output).

**Script structure:**

1. **Configuration constants** at top:
   - `FINETUNED_URL = "http://localhost:8000/v1"` -- vLLM endpoint for fine-tuned model (user overrides via CLI arg)
   - `FINETUNED_MODEL = "chuckles-qwen3-32b-dpo"` -- must match --served-model-name
   - `BASE_URL = ""` -- base model endpoint (optional, user provides via CLI arg)
   - `BASE_MODEL = "Qwen/Qwen3-32B-AWQ"` -- or whatever the base model's served name is
   - `API_KEY = "not-needed"` -- dummy key for vLLM (no auth)

2. **Test prompts** -- a list of 10 movie titles covering diverse phonetic challenges:
   - Mix of short titles (Jaws, Frozen), medium (Pulp Fiction, Fight Club), and long (The Shawshank Redemption, Eternal Sunshine of the Spotless Mind)
   - These are input titles, not full prompts. The script wraps them in a parody generation prompt.

3. **Prompt template** -- match the system prompt and user prompt format used by the `chuckles generate` CLI. Reference the existing `PARODY_INSTRUCTIONS` or `DATASET_SYSTEM_PROMPT` from the codebase to stay consistent. The prompt should ask for a phonetically-sound parody of the given title.

4. **query_model(client, model_name, prompt) -> str** function:
   - Uses `openai.OpenAI(api_key=API_KEY, base_url=url)`
   - Sends chat completion with temperature=0.7, top_p=0.8, max_tokens=512
   - Returns the response text
   - Wraps in try/except, returns "[ERROR: ...]" string on failure

5. **score_parody(original_title, parody_text) -> dict** function:
   - Compute simple quality metrics (these don't need the full phonetic tools -- keep the script standalone):
     a. **Word count ratio**: len(parody.split()) / len(original.split()) -- closer to 1.0 is better (structure preservation)
     b. **Character similarity**: SequenceMatcher(None, original.lower(), parody.lower()).ratio() -- measures overall character overlap
     c. **Non-empty**: 1.0 if parody has content, 0.0 if empty/error
   - Return dict with these three scores

6. **main() function:**
   a. Parse CLI args: `--finetuned-url`, `--base-url` (optional), `--finetuned-model`, `--base-model`
   b. Create OpenAI client(s) for fine-tuned model (and optionally base model)
   c. For each test title:
      - Query fine-tuned model, score the result
      - If base URL provided: query base model, score the result
      - Print side-by-side comparison
   d. Print summary table:
      - Average scores per model across all prompts
      - Per-prompt comparison if both models tested
   e. Print verdict: "PASS" if fine-tuned model has higher average character similarity than base, or if base model not tested, just print fine-tuned model scores

7. **Standalone mode (no base model):**
   - If `--base-url` not provided, only test the fine-tuned model
   - Print scores and example outputs
   - This is the default mode -- user may not have two models running simultaneously

8. **Dependencies comment** at top: `# Requires: pip install openai` (already in project)

IMPORTANT: Keep this script fully standalone -- no imports from chuckles_prime package. It runs on RunPod where the package may not be installed. Only use stdlib + openai.
