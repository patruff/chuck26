  <name>Task 1: Create vLLM inference launch script</name>
  <files>training/setup_inference.sh</files>
  <action>
Create `training/setup_inference.sh` -- a standalone bash script that sets up and launches vLLM on a RunPod GPU pod. Follow the same pattern as `training/setup_runpod.sh` (set -e, clear comments, configuration variables at top).

**Script structure:**

1. **Configuration block** at top:
   - `MODEL_ID="patruff/chuckles-qwen3-32b-dpo"` (FP16 merged model)
   - `AWQ_MODEL_ID="patruff/chuckles-qwen3-32b-dpo-awq"` (pre-quantized AWQ, if available)
   - `SERVED_NAME="chuckles-qwen3-32b-dpo"` (must match settings.json model_name)
   - `PORT=8000`
   - `MAX_MODEL_LEN=8192` (NOT default 40960 -- OOM on all consumer GPUs)
   - `GPU_MEM_UTIL=0.90`

2. **Parse command-line argument** for quantization mode:
   - `--bnb` (default): BitsAndBytes runtime NF4 quantization. Simpler, no pre-quantization needed. ~168 tok/s. Needs ~65GB+ system RAM for loading.
   - `--awq`: Serve pre-quantized AWQ model. Faster (~712 tok/s with Marlin). Requires running quantize_awq.py first.
   - Print usage if `--help` or invalid arg.

3. **Install vLLM** (pip install vllm>=0.15.0). Skip if already installed.

4. **Pre-flight checks:**
   - Check GPU available (`nvidia-smi`)
   - For `--bnb` mode: Check system RAM >= 65GB (`free -g`), warn if less
   - Print selected quantization mode, model ID, served name

5. **Launch vLLM serve** with these CRITICAL flags:
   - `--served-model-name "$SERVED_NAME"` -- CRITICAL: must match settings.json model_name. Without this, vLLM uses full HF repo ID and requests fail with 404.
   - `--max-model-len 8192` -- CRITICAL: default 40960 causes OOM. Parodies need < 8K tokens.
   - `--default-chat-template-kwargs '{"enable_thinking": false}'` -- CRITICAL: without this, Qwen3 produces <think> blocks that break the smolagents parser.
   - `--gpu-memory-utilization 0.90` -- use 90% of VRAM for model + KV cache
   - `--host 0.0.0.0` -- bind to all interfaces (required for RunPod port forwarding)
   - `--port $PORT`
   - `--dtype half` -- explicit FP16 (matches merged model dtype)
   - For `--bnb`: add `--quantization bitsandbytes`
   - For `--awq`: use `$AWQ_MODEL_ID` instead of `$MODEL_ID` (no --quantization flag needed, model is already quantized)

6. **Post-launch instructions** printed before launch:
   - "Server starting on port $PORT..."
   - "Test with: curl http://localhost:$PORT/v1/models"
   - "Update settings.json with: api_base_url = http://<POD_IP>:$PORT/v1"
   - "Set env var: export VLLM_API_KEY='not-needed'"

Make the script executable (chmod +x in the script header comment).

<task type="auto">
  <name>Task 2: Create AWQ quantization script</name>
  <files>training/quantize_awq.py</files>
  <action>
Create `training/quantize_awq.py` -- a standalone Python script that quantizes the FP16 merged model to AWQ 4-bit and pushes to HuggingFace Hub. Follow the same pattern as `training/merge_and_push.py` (configuration constants at top, main() function, clear print statements).

**Script structure:**

1. **Configuration constants** at top:
   - `MODEL_ID = "patruff/chuckles-qwen3-32b-dpo"` -- source FP16 model on Hub
   - `QUANT_OUTPUT = "/workspace/chuckles-qwen3-32b-dpo-awq"` -- local save path (RunPod /workspace/ for persistence)
   - `HUB_REPO = "patruff/chuckles-qwen3-32b-dpo-awq"` -- target Hub repo
   - AWQ config: `zero_point=True, q_group_size=128, w_bit=4, version="GEMM"`

2. **main() function:**
   a. Print "Loading FP16 model from Hub..." and load with `AutoAWQForCausalLM.from_pretrained(MODEL_ID)`
   b. Load tokenizer with `AutoTokenizer.from_pretrained(MODEL_ID)`
   c. Print "Quantizing to AWQ 4-bit..." and call `model.quantize(tokenizer, quant_config=quant_config)`
   d. Print "Saving quantized model locally..." and call `model.save_quantized(QUANT_OUTPUT)` + `tokenizer.save_pretrained(QUANT_OUTPUT)`
   e. Print "Pushing to Hub..." and use `HfApi().upload_folder(folder_path=QUANT_OUTPUT, repo_id=HUB_REPO, repo_type="model")`
   f. Print "Done! Serve with: python setup_inference.sh --awq" (sic -- should say bash setup_inference.sh --awq)

3. **Validation step** (after quantize, before push):
   - Load the quantized model back with `AutoAWQForCausalLM.from_quantized(QUANT_OUTPUT)`
   - Load tokenizer, apply chat template with a simple test prompt (e.g., "Say hello"), generate a short response
   - Print the response to visually verify it's coherent (not garbled)
   - If generation fails or produces empty output, print warning but continue (don't block the push)

4. **Error handling:**
   - Wrap HfApi upload in try/except, print fallback instructions if push fails
   - Check HF_TOKEN is set at script start (same pattern as merge_and_push.py)

5. **Dependencies comment** at top: `# Requires: pip install autoawq transformers huggingface-hub`

NOTE: Use AutoAWQ (not LLM Compressor) for Qwen3-32B quantization. LLM Compressor has known quality issues with Qwen3-32B W4A16 (GitHub issue vllm-project/llm-compressor#1600). AutoAWQ is deprecated but still functional and produces better results.
