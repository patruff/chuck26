  <action>
Create `src/chuckles_prime/rewards.py` with three pure functions that compute continuous reward signals in [0.0, 1.0]:

1. `compute_phonetic_quality(candidate: ParodyCandidate) -> float`:
   - Average of all values in `candidate.phonetic_scores`. Return 0.0 if empty.
   - Import ParodyCandidate from `chuckles_prime.types`.

2. `compute_tool_usage_completeness(trace: AgentTrace, input_title: str) -> float`:
   - Count title words with len > 2. If none, return 1.0.
   - Scan each step in `trace.steps` (stringified) for case-insensitive matches of each title word.
   - Return fraction of title words found in any step.
   - Import AgentTrace from `chuckles_prime.types`.

3. `compute_structure_preservation(input_title: str, parody_text: str) -> float`:
   - Split both on whitespace, compute `min(len_parody, len_orig) / max(len_parody, len_orig)`.
   - Return 0.0 if original is empty.

All three functions are stateless, take simple inputs, return a float. No classes, no side effects.

Also update `pyproject.toml` dependencies array to add `"datasets>=3.0.0"` and `"huggingface-hub>=0.20.0"` after the existing `"pronouncing"` entry.

Create `tests/test_rewards.py` with pytest tests covering:
- `compute_phonetic_quality`: empty scores -> 0.0, single score, multiple scores (verify average)
- `compute_tool_usage_completeness`: no steps -> 0.0 for words, short words skipped, partial matches
- `compute_structure_preservation`: same word count -> 1.0, double word count -> 0.5, empty title -> 0.0

Use the dataclass constructors from types.py directly (ParodyCandidate, AgentTrace) -- no mocking needed for pure data containers.
