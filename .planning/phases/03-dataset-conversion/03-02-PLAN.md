Create `src/chuckles_prime/dataset.py` with these functions:

**Constants:**
```python
DATASET_SYSTEM_PROMPT = (
    "You are a comedy writer who creates funny parody titles. "
    "Replace words with phonetically similar but humorous alternatives. "
    "Use the phonetic analysis tool to verify similarity scores above 0.6."
)
```
Use this simplified system prompt (NOT the full PARODY_INSTRUCTIONS from prompts.py). The dataset system prompt describes the task for training, not the agent workflow.

**1. `records_to_grpo_dataset(records: list[GenerationRecord]) -> Dataset`:**
- Import Dataset from `datasets`.
- Import `compute_phonetic_quality`, `compute_tool_usage_completeness`, `compute_structure_preservation` from `chuckles_prime.rewards`.
- Skip records where `rec.error is not None`.
- For each valid record, build a dict with:
  - `"prompt"`: list of message dicts `[{"role": "system", "content": DATASET_SYSTEM_PROMPT}, {"role": "user", "content": f"Create a phonetically-sound parody of: '{rec.input_title}'"}]`
  - `"original_title"`: `rec.input_title` (string)
  - `"phonetic_scores"`: `json.dumps({c.text: c.phonetic_scores for c in rec.candidates})` -- JSON string, NOT nested dict
  - `"generation_model"`: `rec.model_name` (string)
  - `"avg_phonetic_score"`: average of compute_phonetic_quality across all candidates (float)
  - `"avg_tool_usage"`: compute_tool_usage_completeness(rec.trace, rec.input_title) (float)
  - `"avg_structure_preservation"`: average of compute_structure_preservation(rec.input_title, c.text) for each candidate (float)
- Return `Dataset.from_list(rows)`. If rows is empty, return `Dataset.from_list([])` (empty dataset).

**2. `build_dpo_dataset(human_examples: list[tuple[str, str, str]], model_records: dict[str, GenerationRecord]) -> Dataset`:**
- For each `(input_title, human_output, explanation)` in human_examples:
  - Look up `model_records.get(input_title)` -- skip if not found or no candidates.
  - Select the worst candidate: `min(rec.candidates, key=lambda c: sum(c.phonetic_scores.values()) / max(len(c.phonetic_scores), 1))`.
  - Build a dict with:
    - `"prompt"`: `[{"role": "system", "content": DATASET_SYSTEM_PROMPT}, {"role": "user", "content": f"Create a phonetically-sound parody of: '{input_title}'"}]`
    - `"chosen"`: `[{"role": "assistant", "content": human_output}]` -- assistant-only message
    - `"rejected"`: `[{"role": "assistant", "content": worst.text}]` -- assistant-only message
- Return `Dataset.from_list(rows)`.

**3. `push_dataset(dataset: Dataset, repo_id: str, split: str = "train", private: bool = True) -> None`:**
- Import `os`, `login` from `huggingface_hub`.
- Check `os.environ.get("HF_TOKEN")` -- raise `ValueError` with clear message if not set.
- Call `login(token=token)`.
- Call `dataset.push_to_hub(repo_id, split=split, private=private)`.

**Key anti-patterns to AVOID (from research):**
- Do NOT include model completions in GRPO rows (prompt-only).
- Do NOT store nested Python dicts as dataset columns -- use json.dumps for phonetic_scores.
- Do NOT use the full PARODY_INSTRUCTIONS as system prompt -- use simplified DATASET_SYSTEM_PROMPT.
- chosen/rejected in DPO must be assistant-only messages (do NOT repeat the user prompt).
