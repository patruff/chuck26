# Plan 04-01: Pipeline CLI Entry Point

## Goal

Create the CLI module (`cli.py`) and its tests (`test_cli.py`) so that users can run `chuckles generate input.csv` for the full pipeline (read CSV, generate parodies, archive traces, build GRPO/DPO datasets, push to Hub) or `chuckles convert traces.jsonl` to produce datasets from previously saved traces -- both with rich progress output and a summary table on completion.

## Context

All pipeline modules exist and are tested (config, model, tools, generator, traces, dataset, rewards). The entry point `chuckles = "chuckles_prime.cli:main"` is already declared in pyproject.toml. Research (04-RESEARCH.md) recommends argparse with subcommands, lazy imports inside command handlers, rich Progress for per-title tracking, and a hand-rolled `_record_from_dict()` for JSONL deserialization (~20 lines). No new dependencies are needed.

Key function signatures to wire together:
- `load_config(settings_path) -> AppConfig`
- `read_input_titles(csv_path) -> list[str]`
- `create_model(config) -> OpenAICompatibleModel`
- `load_parody_tools() -> (parody_tool, phone_tool)`
- `create_agent(model, parody_tool, phone_tool) -> CodeAgent`
- `generate_single(title, agent, parody_tool, config) -> GenerationRecord`
- `archive_traces(records, output_path) -> int`
- `records_to_grpo_dataset(records) -> Dataset`
- `build_dpo_dataset(human_examples, model_records) -> Dataset`
- `push_dataset(dataset, repo_id, split, private) -> None`

Key types for JSONL deserialization:
- `GenerationRecord(input_title, candidates, trace, model_name, error)`
- `ParodyCandidate(text, phonetic_scores, humor_note)`
- `AgentTrace(steps, final_output, token_usage, state)`

## Tasks

### Task 1: Create `src/chuckles_prime/cli.py`
**File:** `src/chuckles_prime/cli.py`
**Action:** Create

Create the CLI module with the following structure. All heavy imports (config, model, tools, generator, traces, dataset) must be lazy -- placed inside command handler functions, not at module top level. Only stdlib imports (argparse, sys, json, pathlib) and rich imports (Console, Progress, Table, and column types) go at the top level.

**`main()` function:**
- Create an `argparse.ArgumentParser` with `prog="chuckles"` and description `"Generate phonetically sound parody titles and RLVR datasets"`.
- Add subparsers with `dest="command"` (do NOT set `required=True` -- handle missing command manually to show help).
- Register two subcommands: `generate` and `convert` (details below).
- If `args.command is None`, print help and `sys.exit(1)`.
- Wrap `args.func(args)` in try/except: catch `KeyboardInterrupt` (print `"[yellow]Interrupted[/yellow]"`, exit 130), catch `Exception` (print `"[bold red]Error:[/bold red] {e}"`, exit 1).

**`generate` subcommand arguments:**
- `input` (positional): Path to input CSV with 'title' column
- `--settings` (default `"settings.json"`): Path to settings JSON
- `--output-dir` (default `"output"`): Directory for traces JSONL
- `--grpo-repo` (default `None`): HF Hub repo ID for GRPO dataset
- `--dpo-repo` (default `None`): HF Hub repo ID for DPO dataset
- `--no-push` (`action="store_true"`): Skip pushing to Hub
- Set defaults: `func=cmd_generate`

**`convert` subcommand arguments:**
- `traces` (positional): Path to traces JSONL file
- `--settings` (default `"settings.json"`): Path to settings JSON (needed for DPO human examples)
- `--grpo-repo` (default `None`): HF Hub repo ID for GRPO dataset
- `--dpo-repo` (default `None`): HF Hub repo ID for DPO dataset
- `--no-push` (`action="store_true"`): Skip pushing to Hub
- Set defaults: `func=cmd_convert`

**`cmd_generate(args)` function -- full pipeline:**
1. Lazy import: `load_config`, `create_model`, `load_parody_tools`, `read_input_titles`, `create_agent`, `generate_single`, `archive_traces`, `records_to_grpo_dataset`, `build_dpo_dataset`, `push_dataset`, and `GenerationRecord`, `AgentTrace`, `ParodyCandidate` from types.
2. Load config: `config = load_config(args.settings)`.
3. Read titles: `titles = read_input_titles(args.input)`. Print count: `"Loaded {len(titles)} titles from {args.input}"`.
4. Create model, load tools, create agent.
5. Generate with rich Progress (TextColumn, BarColumn, MofNCompleteColumn, TimeElapsedColumn). Loop over titles calling `generate_single()`. Wrap each call in try/except -- on exception, create a fallback `GenerationRecord` with `error=str(e)`, empty candidates, and an AgentTrace with `state="error"`. After each title, print status via `progress.console.print()`: green "OK" or red "ERR: {error}". Call `progress.advance(task)`.
6. Archive traces: `output_dir = Path(args.output_dir)`, `output_dir.mkdir(parents=True, exist_ok=True)`, `traces_path = output_dir / "traces.jsonl"`, `count = archive_traces(records, traces_path)`. Print count.
7. Build GRPO dataset: `grpo_ds = records_to_grpo_dataset(records)`. Print row count.
8. Build DPO dataset: `model_records = {r.input_title: r for r in records if not r.error}`, then `dpo_ds = build_dpo_dataset(config.human_examples, model_records)`. Print row count.
9. Push datasets if not `args.no_push`:
   - If `args.grpo_repo`: call `push_dataset(grpo_ds, args.grpo_repo)`, print confirmation.
   - If `args.dpo_repo`: call `push_dataset(dpo_ds, args.dpo_repo)`, print confirmation.
   - If neither repo specified and push not disabled, print a note: `"No --grpo-repo or --dpo-repo specified; skipping Hub push"`.
10. Call `_print_summary(records, console)`.

**`cmd_convert(args)` function -- dataset-only from JSONL:**
1. Lazy import: `load_config`, `records_to_grpo_dataset`, `build_dpo_dataset`, `push_dataset`.
2. Validate traces file exists: `traces_path = Path(args.traces)`, if not exists raise `FileNotFoundError(f"Traces file not found: {traces_path}")`.
3. Load records: `records = load_records(traces_path)`. Print count: `"Loaded {len(records)} records from {traces_path}"`.
4. Load config: `config = load_config(args.settings)` (needed for DPO human examples).
5. Build GRPO and DPO datasets (same logic as cmd_generate steps 7-8).
6. Push datasets (same logic as cmd_generate step 9).
7. Call `_print_summary(records, console)`.

**`_record_from_dict(d: dict) -> GenerationRecord` helper:**
Reconstruct a GenerationRecord from a dict (as produced by `dataclasses.asdict()` + `json.dumps()`). Build nested `ParodyCandidate` list from `d["candidates"]` and `AgentTrace` from `d["trace"]`. Use `.get()` with sensible defaults for optional fields: `phonetic_scores` defaults to `{}`, `humor_note` to `""`, `token_usage` to `None`, `state` to `"unknown"`, `error` to `None`, `model_name` to `"unknown"`. See research code example for exact implementation.

**`load_records(jsonl_path: Path) -> list[GenerationRecord]` function:**
Open the JSONL file, read line by line, skip empty lines, `json.loads()` each line, pass to `_record_from_dict()`, collect into list and return.

**`_print_summary(records, console)` function:**
Create a `rich.table.Table` with title `"Generation Summary"`. Two columns: "Metric" (bold) and "Value" (right-justified). Rows:
- "Titles processed" -> `str(len(records))`
- "Successful" -> green count of records where `error is None`
- "Failed" -> red count (or plain "0" if none failed)
- "Total candidates" -> sum of `len(r.candidates)` across all records

Print the table via `console.print(table)`.

**Module-level:**
- `console = Console(stderr=True)` -- send progress/status to stderr so stdout stays clean for piping.

### Task 2: Create `tests/test_cli.py`

**Implementation note for Task 2:** Since `cmd_generate` uses lazy imports (importing inside the function body), the monkeypatch approach in test 11 needs to patch the names AFTER they are imported into the cli module's local scope. The cleanest approach: in `cmd_generate`, after the lazy imports, the function uses local names. To make these patchable, assign them to module-level names or use the pattern where `monkeypatch.setattr("chuckles_prime.config.load_config", mock_fn)` patches at the source module level. The test should patch at the source module level (e.g., `monkeypatch.setattr("chuckles_prime.config.load_config", ...)`) rather than on the cli module. However, since imports are lazy and local, the more reliable approach is to have `cmd_generate` import via module references: `from chuckles_prime import config` then call `config.load_config(...)`. This lets tests patch `chuckles_prime.config.load_config`. **Use this module-reference import pattern in cli.py** so that lazy imports remain patchable.

Alternative: If the module-reference pattern is too verbose, factor the pipeline steps into a `_run_generate(...)` function that accepts callables, and have `cmd_generate` pass the real functions. Tests can then call `_run_generate` with mocks. Choose whichever approach is simpler.

## Notes
- **Lazy imports are critical.** `chuckles --help` must be fast. smolagents, datasets, openai, and torch are heavy. Only import them inside `cmd_generate` and `cmd_convert`.
- **CLI calls `generate_single()` in a loop**, not `generate_batch()`. This decouples progress display from generation logic and avoids the bare `print()` calls in `generate_batch()`.
- **`_build_parser()` factored out** from `main()` to make argument parsing testable without invoking `sys.exit`.
- **Console uses `stderr=True`** so progress output goes to stderr, keeping stdout clean for potential piping.
- **Hub push is opt-in by repo flags.** No default repo IDs. If `--no-push` is set, skip entirely. If no repo flags given and push not disabled, print a note but don't error.
- **`build_dpo_dataset` needs `config.human_examples`**, which is why `--settings` is accepted by both subcommands. The convert command needs settings to build DPO pairs.
- **Error isolation per title:** In the generate loop, each `generate_single()` call is wrapped in try/except. On exception, a fallback GenerationRecord is created with the error message. The loop continues to the next title.
